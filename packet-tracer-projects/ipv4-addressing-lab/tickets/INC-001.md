## Documentation Summary

**Root Cause:**  Default Gateway not configured on the PCs  
**Resolution:**  Configured the Default Gateway on each PCs designated router IP  
**Time to Resolve:**  ~1 hour  
**Lessons Learned:**  
- When configuring IP on each PC make sure to see the Default Gateway  
- When troubleshooting inter network ping 100% packet loss check the PC's IP configuration
- The first packet timeout is normal on cross-network pings. The ARP resolution learning MAC addresses of the default gateway.
- 100% packet loss means theres a complete break in the connection between the source and the network. No data reached the target. A common problem is network misconfigurations
